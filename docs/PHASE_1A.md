# Phase 1A Implementation Note

Phase 1A introduces the first concrete CairnGraph pipeline.

Input:

- CairnStone chain manifest

Processing:

- Normalize chain into a CairnGraph model
- Resolve chain node
- Resolve HEAD node
- Resolve stone nodes
- Preserve grounded CairnStone graph edges
- Track unresolved manifest references

Output:

- Mermaid flowchart renderer
- Test fixture for loop-engineer-template-review
- Node test coverage

Validation:

- TypeScript build via tsc
- Node test runner
- GitHub Actions CI scaffold

Known follow-up:

- Replace the compact fixture with a full manifest fixture generated directly from CairnStone V5.
- Add ref-level nodes from stone LOD2 compressed indexes.
- Add source URL and GitHub commit navigation slots.
- Add package lockfile generated by npm instead of a placeholder lockfile.
